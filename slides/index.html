Begin with the WHY

Why should we move from Vibe Coding to Agent Engineering?

💸 1. The Reliability Why

Mitigating "The Hidden Tax" of Vibe Coding

📈 1.7x more major issues

🔀 75% more logic errors (flawed control flow)

🔓 2.74x higher rate of security vulnerabilities

Source: CodeRabbit analysis of AI co-authored code without rigorous discipline

🎯 Goal: Turn unreliable agents into reliable systems through testing and oversight

🎼 2. The Professional Why

Orchestration vs. Consumption

❌ Vibe Coding

"Forget that the code even exists and just embrace the vibes"

✅ Agent Engineering

Act as oversight for agents that write code

🎯 Goal: Move from "prompt consumers" to "agent engineers" who lead a team of digital workers

⚠️ 3. The Risk Why

Avoiding the "Plausible but Wrong" Trap

📋 Business Analysts

Agents might generate requirements that are "plausible but wrong"

🏗️ Solution Architects

Agents may "hallucinate" design patterns that don't fit

👨‍💻 Developers

Code that works but introduces subtle bugs or security holes

🧪 QA Engineers

Tests that pass but don't actually validate requirements

🎯 Goal: Use engineering discipline (Plan, Review, Test, Own) to ensure we don't deploy code we can't explain or verify

📈 4. The Scale Why

Moving Toward Autonomy (The 4 Modes Framework)

Mode 4: Designing – Agent architects solutions
Mode 3: Delegating – Agent handles features
Mode 2: Directing – Agent completes tasks
Mode 1: Doing – Agent assists with snippets

⚠️ Without guardrails, asking an agent to "add docstrings" might refactor your entire codebase

🎯 Goal: Establish guardrails and "kill switches" so humans remain in control of ethics and constraints

History of Context-Based LLM Coding

From autocomplete to autonomous agents (2023-2026)

📅 The Evolution (2023-2024)

2023

Context-Unaware Code Snippets

LLMs as "supercharged autocomplete" • File-level suggestions only
No repository indexing • Human integrates, refactors, tests everything
Tools: Early GitHub Copilot, Codex

2024

Context-Aware Code Generation

Models read multiple files • Repository-wide semantic search
Multi-step prompts • RAG for code (API docs, internal libraries)
Capability: "Write tests for this function"

📅 The Evolution (2025-2026)

2025

Single Agents for Atomic Coding Tasks

LLM as a code agent that can plan and act, not just reply
Decomposes tasks, calls tools (compilers, linters, test runners)
Example: "Implement GET /products and validate with curl"

2026

Parallel Agents for Large-Scale Work

Multi-agent workflows • Repo-scale changes
Concurrent workers with git worktrees • Validation agents
Example: "Migrate our app from Redux to React Query"

🎸 What is "Vibe Coding"?

Unstructured, trial-and-error prompting with AI
"Just ask and hope for the best"

  • Vague prompts: "Make it work" or "Fix the bug"
  • No context provided to the AI
  • Random iteration without a plan
  • Copy-paste without understanding

🔧 What is "Agent Engineering"?

Systematic, context-aware development with AI agents
"Precise instructions with clear outcomes"

  • Specific, actionable prompts
  • Provide file paths, line numbers, context
  • Break tasks into manageable chunks
  • Iterative refinement with feedback

An open-source platform for building AI software development agents

❌ Bad Prompts (Vibe Coding)

❌ Examples of What NOT to Do

"Make the code better"
"Fix the bug"
"Rewrite the entire backend"
"There's a bug somewhere in auth"

⚠️ Too vague • No location • Not scoped • No context

✅ Good Prompts (Agent Engineering)

✅ The Three Principles

  • Concrete – Describe specific functionality
  • Location-specific – Include file paths & line numbers
  • Appropriately scoped – ~100 lines, single feature

✅ Good Prompt Examples

"Add a function `calculate_average` in `utils/math_operations.py` that takes a list of numbers and returns their average."
"Fix the TypeError in `frontend/src/components/UserProfile.tsx` on line 42. The error says we're accessing a property of undefined."
"Implement email validation in `RegistrationForm.tsx` to check format before submission."

👋 Hello World Agent

from openhands.sdk import LLM, Agent, Conversation, Tool
from openhands.tools.terminal import TerminalTool
from openhands.tools.file_editor import FileEditorTool

# Configure LLM
llm = LLM(
    model="anthropic/claude-sonnet-4-5-20250929",
    api_key=os.getenv("LLM_API_KEY"),
)

# Create agent with tools
agent = Agent(llm=llm, tools=[
    Tool(name=TerminalTool.name),
    Tool(name=FileEditorTool.name),
])

# Run!
conversation = Conversation(agent=agent, workspace=os.getcwd())
conversation.send_message("Write 3 facts about this project to FACTS.txt")
conversation.run()

💡 Best Practices

  • Start small – Begin with simple tasks, iterate
  • Be specific – Include file paths, line numbers, context
  • Break down tasks – One feature per prompt (~100 lines)
  • Commit frequently – Version control after each success
  • Include errors – Share full error messages and logs
